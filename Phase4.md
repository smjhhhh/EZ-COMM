# AI旅行Agent - Phase 4 任务图 (Week 25-32)

核心目标： 规模化增长 (100 -> 1000+ 用户)，加深认知资产护城河，优化付费转化率。

核心交付物： 1. 正式版 V1 产品 (Web) 2. 规模化的“认知资产” (KG > 5000+ 条目) 3. 可持续的增长渠道

核心原则： 1. 数据驱动增长 2. 自动化资产沉淀 3. 优化单位经济效益 (LTV/CAC)

### Sprint 13 (Week 25-26): 公测复盘 & 增长启动

**目标：** 深入分析 Phase 3 的公测数据，确定V1正式版的优化方向，并启动首个增长飞轮。

| **任务ID** | **任务类型** | **任务描述**           | **关键交付物**              | **备注**                                                         |
| ---------------- | ------------------ | ---------------------------- | --------------------------------- | ---------------------------------------------------------------------- |
| **P4.T1**  | `Product`        | **公测数据深度分析**   | - Phase 3 (100+用户) 数据分析报告 | (遵照 P3.T14) 深入分析：用户留存曲线、付费转化漏斗、最高频使用场景。   |
| **P4.T2**  | `Growth`         | **确定V1增长渠道**     | - V1 增长策略 (Content-Led)       | 识别 Phase 3 中最有效的 PLG 渠道 (e.g., 来自专业用户的推荐)。          |
| **P4.T3**  | `Product`        | **V1 定价 & 付费优化** | - V1 正式版定价 (e.g., 月/年订阅) | 基于 `P3.T14`的 `5%+`转化率数据，A/B 测试新的价格点和功能包。      |
| **P4.T4**  | `Team`           | **启动招聘 (Hire #1)** | - 1-2个 招聘岗位描述 (JD)         | 优先招聘：1. 增长/运营 (Growth/Community) 2. 全栈工程师 (Full-stack)。 |

### Sprint 14 (Week 27-28): 护城河加深 - 资产自动化

**目标：** 将“认知资产”的沉淀从“半自动”升级为“全自动”，实现规模化数据飞轮。

| **任务ID** | **任务类型** | **任务描述**            | **关键交付物**                | **备注**                                                                                                                                                                                                      |
| ---------------- | ------------------ | ----------------------------- | ----------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **P4.T5**  | `Data`           | **KG 知识提取全自动化** | - 自动化 Knowledge Pipeline v1      | (升级 P3.T8) [核心护城河] 创建一个独立服务：1. 实时监控 user_feedback 表2. LLM 自动提取 "Tip" (e.g., "这家店下午5点关门")3. 自动进行事实核查 (调用Google搜索)4. 自动写入 poi_kg (高置信度) 或 "待审核" (低置信度)。 |
| **P4.T6**  | `Infra`          | **内部工具 (Admin)**    | - V1 内部管理后台                   | 为 P4.T4 招聘的新员工准备。功能：1. 用户查询2. 审核 P4.T5 提交的 "待审核" 知识。                                                                                                                                    |
| **P4.T7**  | `Test`           | **评估集规模化**        | -`golden_set_v2.json`(500+ Cases) | 将 Phase 3 公测中所有典型的失败 Case 和边缘 Case 全部加入评估集。                                                                                                                                                   |
| **P4.T8**  | `DevOps`         | **生产环境部署**        | - 生产环境 (Cloud Run / Fargate)    | (完成 P3.T9) 正式将所有服务迁移到可扩展的生产环境。                                                                                                                                                                 |

| **任务ID** | **任务类型** | **任务描述**            | **关键交付物**                           | **备注**                                                                              |
| ---------------- | ------------------ | ----------------------------- | ---------------------------------------------- | ------------------------------------------------------------------------------------------- |
| **P4.T9**  | `Frontend`       | **V1 正式版 (Next.js)** | - V1 Web App (Public)                          | 修复公测版所有P0/P1 Bug，UI/UX 优化，付费流程跑通。                                         |
