# AI旅行Agent - Phase 5 任务图 (Week 33-40)

核心目标： 执行V2产品线扩张（如：7天+长途规划），升级Agent架构以应对高复杂度，并启动B2B市场探索。

核心交付物： 1. V2 (长途规划) Alpha版 2. V2版Agent架构（层级规划） 3. B2B SaaS 市场验证报告

核心原则： 1. 复用资产（KG, 评估） 2. 架构先行（应对复杂度） 3. 多线探索（V1增长, V2研发, B2B探索）

### Sprint 17 (Week 33-34): V2 启动 & 架构升级

**目标：** 最终确定V2扩张方向，重构Agent架构以支持长途、多城规划，并继续扩大团队。

| **任务ID** | **任务类型** | **任务描述**               | **关键交付物**   | **备注**                                                                                                                                                                                                           |
| ---------------- | ------------------ | -------------------------------- | ---------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **P5.T1**  | `Product`        | **V2 战略方向拍板**        | - V2 产品规格书 (PRD)  | (遵照 P4.T16) 假设拍板：**V2 = 7天+长途/多城规划** 。这需要全新的Agent逻辑。                                                                                                                                       |
| **P5.T2**  | `Team`           | **团队扩张 (Hire #2, #3)** | - 2个新成员 Onboarded  | (遵照 P4.T4) 招聘：1. 资深后端/Agent工程师 2. 产品设计师 (PD/UX)。                                                                                                                                                       |
| **P5.T3**  | `Tech`/`LLM`   | **Agent 架构 V2 - 设计**   | - V2 架构设计文档      | [核心技术] 短途规划的Agent无法处理7天行程。必须升级为**“层级规划器” (Hierarchical Planner)**。1. MasterPlanner (负责7天主题、城市分配)2. DailyPlanner (负责每天的具体行程)3. LogisticsAgent (负责机票/酒店/交通衔接)。 |
| **P5.T4**  | `Growth`         | **V1 增长 & 留存优化**     | - V1 付费转化率提升 5% | V1 不能停。`P4.T4`招聘的Growth负责人必须背V1的增长指标。A/B 测试 V1 定价。                                                                                                                                             |

**目标：** 搭建V2架构的骨架，集成V2所需的关键工具（机票/酒店），并并行启动B2B市场探索。

| **任务ID** | **任务类型**  | **任务描述**              | **关键交付物**                                               | **备注**                                                                                      |
| ---------------- | ------------------- | ------------------------------- | ------------------------------------------------------------------ | --------------------------------------------------------------------------------------------------- |
| **P5.T5**  | `Backend`/`LLM` | **V2 架构 - 实施 (v0.1)** | -`MasterPlanner`/`DailyPlanner`/`LogisticsAgent`的 v0.1 代码 | (执行 P5.T3) 搭建层级规划器的代码框架，确保它们可以相互调用。                                       |
| **P5.T6**  | `Backend`         | **V2 工具集成**           | - 机票 API (Skyscanner/携程) 接口- 酒店 API (Booking/Expedia) 接口 | 这是V2的核心能力。**必须是付费API** 。                                                        |

| **P5.T8**  | `PM`              | **B2B 市场探索**          | - 10个旅行社/定制游公司访谈纪要                                    | (并行线程)`P4.T16`的另一个方向。PM开始验证：旅行社是否愿意为我们的Agent能力付费 (SaaS)？          |

**目标：** V2 长途规划功能 E2E (端到端) 跑通 (内部Alpha版)，并扩展知识图谱 (KG) 以支持新工具。

| **任务ID** | **任务类型**  | **任务描述**             | **关键交付物**                        | **备注**                                                                                                        |
| ---------------- | ------------------- | ------------------------------ | ------------------------------------------- | --------------------------------------------------------------------------------------------------------------------- |
| **P5.T9**  | `Frontend`        | **V2 前端 UI/UX**        | - V2 长途规划 (多日/多城) 设计稿 & 前端页面 | (遵照 P5.T2) 新招聘的设计师和前端合作，设计全新的、支持复杂规划的UI (e.g., 可拖拽日程、地图航班显示)。                |
| **P5.T10** | `Backend`/`LLM` | **V2 E2E Alpha (内部)**  | - 可生成7天行程的内部API                    | 将 `P5.T5`和 `P5.T6`组合，实现：输入"巴黎7日游" -> 输出包含(模拟)航班、酒店和每日POI的JSON。                      |
